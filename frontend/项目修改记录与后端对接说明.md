## 三、后端接口对接说明

### 3.1 用户相关接口（src/api/user.js）

#### 3.1.1 验证码接口
```javascript
// 前端调用
getCaptcha()

// 后端接口
GET /user/captcha

// 说明：
// - 返回验证码图片或验证码ID
// - 用于登录和注册时的验证
```

#### 3.1.2 用户注册接口
```javascript
// 前端调用
userRegisterService(registerData)

// 后端接口
POST /user/register

// 请求参数：
{
  username: string,    // 用户名
  password: string,    // 密码
  phone?: string,      // 手机号（可选）
  email?: string       // 邮箱（可选）
}

// 说明：
// - 注册成功后返回token
// - token需要存储在localStorage中
```

#### 3.1.3 用户登录接口
```javascript
// 前端调用
userLoginService(loginData)

// 后端接口
POST /user/login

// 请求参数：
{
  username: string,    // 用户名
  password: string,    // 密码
  captcha?: string     // 验证码（如果需要）
}

// 返回数据：
{
  token: string,      // JWT token
  userId: number,     // 用户ID
  // ...其他用户信息
}

// 说明：
// - token需要存储在localStorage中
// - token格式：JWT，包含userId在payload中
```

#### 3.1.4 获取当前用户信息接口
```javascript
// 前端调用
getCurrentUserInfo()

// 后端接口
GET /user/currentUserInfo

// 请求头：
Authorization: Bearer {token}

// 返回数据：
{
  userId: number,
  username: string,
  phone?: string,
  email?: string,
  // ...其他用户信息
}

// 说明：
// - 需要验证token有效性
// - 返回当前登录用户的详细信息
```

#### 3.1.5 更新用户信息接口

### 3.2 AI相关接口（src/api/user.js）

#### 3.2.1 AI聊天接口（SSE流式响应）
```javascript
// 前端调用
chatService(chatData)

// 后端接口
POST /api/ai/chat

// 请求头：
Authorization: Bearer {token}
Content-Type: multipart/form-data

// 请求参数（FormData）：
{
  chatId: string,      // 聊天会话ID
  prompt?: string,     // 文本问题
  images?: File[]      // 图片文件（可选）
}

// 返回格式（SSE流）：
data: { content: string }  // 或直接返回字符串
data: [DONE]              // 结束标记

// 说明：
// - 使用Server-Sent Events (SSE)实现流式响应
// - 支持文本和图片输入
// - chatId用于关联同一会话的多轮对话
```

#### 3.2.2 获取AI历史记录列表接口
```javascript
// 前端调用
chatHistoryService(type = 'chat')

// 后端接口
GET /ai/history/{type}/list

// 请求头：
Authorization: Bearer {token}

// 路径参数：
type: string  // 类型，默认为'chat'

// 返回数据：
[
  {
    chatId: string,
    title: string,
    createdAt: string,
    updatedAt: string,
    // ...其他字段
  }
]

// 说明：
// - 返回指定类型的聊天会话列表
// - 需要验证token有效性
```

#### 3.2.3 获取单个会话历史接口
```javascript
// 前端调用
getSingleChatHistory(type, chatId)

// 后端接口
GET /ai/history/{type}/{chatId}

// 请求头：
Authorization: Bearer {token}

// 路径参数：
type: string   // 类型
chatId: string // 聊天会话ID

// 返回数据：
{
  chatId: string,
  messages: [
    {
      role: 'user' | 'assistant',
      content: string,
      timestamp: string,
      // ...其他字段
    }
  ],
  // ...其他字段
}

// 说明：
// - 返回指定会话的详细对话历史
// - 需要验证token有效性
```

### 4.3 Token认证说明

#### JWT Token格式要求：
```json
{
  "header": {
    "alg": "HS256",
    "typ": "JWT"
  },
  "payload": {
    "userId": number,    // 用户ID（必需）
    "username": string,  // 用户名
    "iat": number,       // 签发时间
    "exp": number        // 过期时间
  }
}
```

#### Token使用方式：
1. 登录成功后，后端返回token
2. 前端将token存储在localStorage中
3. 后续请求在请求头中携带token：
   ```
   Authorization: Bearer {token}
   ```

### 4.4 SSE流式响应实现说明

#### 后端实现要求：
1. 响应头设置：
   ```
   Content-Type: text/event-stream
   Cache-Control: no-cache
   Connection: keep-alive
   ```

2. 数据格式：
   ```
   data: {"content": "部分回复内容"}

   data: {"content": "更多内容"}

   data: [DONE]
   ```

3. 前端处理：
   - 使用SSEHandler处理流式响应
   - 实时追加AI回复内容
   - 收到[DONE]标记时结束
