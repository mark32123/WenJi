## 一、项目概述

### 1.1 项目背景

在"文化自信"与"数字中国"战略推动下，非遗文化的年轻化传播亟需创新载体。传统展示方式缺乏互动性与激励机制，难以吸引Z世代用户深度参与。"文迹"App以"行走+体验+奖励"为核心，尝试用轻量化技术手段降低文化接触门槛。

### 1.2 项目目标

- 打造一个基于地理位置的传统文化探索工具
- 通过AR虚拟展示增强用户对非遗项目的直观理解
- 建立"打卡—积累—兑换"闭环，提升用户持续使用意愿
- 为后续接入文旅资源、教育场景打下产品基础

### 1.3 核心创新点

| 创新点 | 描述 |
|--------|------|
| 文化地图点亮机制 | 用户每到访一个文化地标即可点亮，形成个人文化足迹 |
| AR沉浸式展示 | 无需专业设备，手机即可查看非遗技艺的3D还原 |
| 实用型激励体系 | 通过体验行为赚取"阅历"，可兑换现实奖励 |
| AI智能讲解 | 集成大语言模型，为用户提供个性化的文物讲解服务 |

## 二、用户界面设计

### 2.1 应用结构

```
文迹 App
├── 山河卷（首页）
│   ├── 地图展示
│   ├── 景点标记
│   ├── 城市切换
│   └── AI助手入口
├── AR鉴赏
│   ├── 扫描模式
│   ├── QR码模式
│   └── AR内容展示
├── 藏经阁（个人中心）
│   ├── 用户信息
│   ├── 收藏列表
│   ├── 徽章展示
│   └── 设置管理
└── 得趣（文物详情）
    ├── 文物卡片
    ├── AI赋诗
    └── 收藏功能
```

### 2.2 主要界面设计

#### 2.2.1 山河卷（首页）

**功能描述**：展示文化遗产地图，用户可浏览景点、查看详情、与AI助手对话。

**界面元素**：
- 顶部：标题栏、阅历显示、AI助手按钮
- 中部：高德地图（景点标记、城市切换按钮）
- 底部：导航栏（山河卷、AR鉴赏、藏经阁）

**交互流程**：
```
用户进入 → 地图加载 → 显示景点标记 → 点击标记 → 弹出景点卡片 → 点击探索 → 显示文物详情
```

#### 2.2.2 AR鉴赏页面

**功能描述**：通过扫描识别图或QR码触发AR体验。

**界面元素**：
- 模式切换按钮（扫描模式/QR码模式）
- 相机预览区域
- AR内容叠加层
- 返回按钮

**交互流程**：
```
选择模式 → 开启相机 → 扫描识别图/QR码 → 加载AR内容 → 显示3D模型/信息
```

#### 2.2.3 藏经阁（个人中心）

**功能描述**：展示用户信息、收藏文物、徽章成就。

**界面元素**：
- 用户头像、昵称、阅历等级
- 收藏文物列表
- 徽章展示区
- 功能入口（编辑资料、设置等）

#### 2.2.4 得趣页面（文物详情）

**功能描述**：展示文物收藏卡片，支持AI赋诗功能。

**界面元素**：
- 文物图片展示区
- 文物名称、朝代、地点
- 诗词展示区
- AI赋诗按钮
- 收藏/保存按钮

### 4.4 核心功能模块设计

#### 4.4.1 用户认证模块

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   用户登录    │────▶│  JWT Token   │────▶│  Redis缓存   │
│   请求       │     │    生成      │     │  用户信息    │
└──────────────┘     └──────────────┘     └──────────────┘
       │                    │                    │
       │                    │                    │
       ▼                    ▼                    ▼
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  验证码校验   │     │  Token返回   │     │  过期管理    │
│  密码校验    │     │  给客户端    │     │  自动续期    │
└──────────────┘     └──────────────┘     └──────────────┘
```

**认证流程**：
1. 用户提交登录表单（用户名、密码、验证码）
2. 服务端验证验证码（Redis缓存）
3. 验证用户名密码（数据库查询）
4. 生成JWT Token
5. 将用户信息存入Redis
6. 返回Token给客户端

## 六、数据流转及展示方式

### 6.1 用户认证数据流

```
┌─────────────────────────────────────────────────────────────────┐
│                        用户认证流程                              │
└─────────────────────────────────────────────────────────────────┘

1. 登录请求
   ┌─────────┐    POST /api/user/login     ┌─────────┐
   │  前端   │ ──────────────────────────▶ │  后端   │
   │         │  {username, password,       │         │
   │         │   captcha, captchaKey}      │         │
   └─────────┘                              └─────────┘

2. 验证码校验
   ┌─────────┐    GET captcha:xxx          ┌─────────┐
   │  后端   │ ──────────────────────────▶ │  Redis  │
   │         │                              │         │
   └─────────┘ ◀────────────────────────── │         │
                验证码值                    └─────────┘

3. 用户查询
   ┌─────────┐    SELECT * FROM user       ┌─────────┐
   │  后端   │ ──────────────────────────▶ │  MySQL  │
   │         │    WHERE username = ?       │         │
   └─────────┘                              └─────────┘

4. Token生成与存储
   ┌─────────┐    生成JWT Token            ┌─────────┐
   │  后端   │ ──────────────────────────▶ │  Redis  │
   │         │    HSET user:login:token    │         │
   │         │    {userId, username, ...}  │         │
   └─────────┘                              └─────────┘

5. 响应返回
   ┌─────────┐    {userId, accessToken}    ┌─────────┐
   │  后端   │ ──────────────────────────▶ │  前端   │
   │         │                              │         │
   └─────────┘                              └─────────┘

6. 前端存储
   ┌─────────┐    存储到 Pinia Store       ┌─────────┐
   │  前端   │ ──────────────────────────▶ │  Pinia  │
   │         │    + localStorage           │         │
   └─────────┘                              └─────────┘
```

```
┌─────────────────────────────────────────────────────────────────┐
│                        地图数据流程                              │

   ┌─────────┐    GET /api/heritage/nearby ┌─────────┐
   │  前端   │ ──────────────────────────▶ │  后端   │
   │         │    ?lng=xxx&lat=xxx         │         │
   └─────────┘                              └─────────┘
                                             │
                                             ▼
                                          ┌─────────┐
                                          │  MySQL  │
                                          │ 查询景点 │
                                          └─────────┘

3. 景点数据返回
   ┌─────────┐    [{siteId, name,         ┌─────────┐
   │  后端   │ ──────────────────────────▶ │  前端   │
   │         │      longitude, latitude,  │         │
   │         │      category, ...}]       │         │

4. 渲染景点标记
   ┌─────────┐    创建Marker实例           ┌─────────┐

6. 彩蛋检测

   │         │    < 2km触发彩蛋            │         │
   └─────────┘                              └─────────┘
```

### 6.3 AR体验数据流

```
┌─────────────────────────────────────────────────────────────────┐
│                        AR体验流程                                │
└─────────────────────────────────────────────────────────────────┘

1. 进入AR页面
   ┌─────────┐    加载MindAR脚本          ┌─────────┐
   │ ArView  │ ──────────────────────────▶ │ CDN/本地│
   │         │    加载A-Frame框架          │         │
   └─────────┘                              └─────────┘

2. 选择扫描模式
   ┌─────────┐                             ┌─────────┐
   │  用户   │ ── 扫描模式 ──────────────▶ │ 相机预览│
   │         │ ── QR码模式 ──────────────▶ │ QR扫描  │
   └─────────┘                              └─────────┘

3. 图像识别（扫描模式）
   ┌─────────┐    加载 .mind 文件          ┌─────────┐
   │ MindAR  │ ──────────────────────────▶ │ /targets│
   │         │    匹配识别目标             │         │
   └─────────┘                              └─────────┘

4. QR码解析（QR码模式）
   ┌─────────┐    解析QR码内容             ┌─────────┐
   │ JSQR    │ ──────────────────────────▶ │ 文物ID  │
   │         │    提取exhibitId            │         │
   └─────────┘                              └─────────┘

5. 加载AR内容
   ┌─────────┐    加载exhibits.json        ┌─────────┐
   │  前端   │ ──────────────────────────▶ │ 静态资源│
   │         │    获取文物配置             │         │
   └─────────┘                              └─────────┘

6. 渲染AR内容
   ┌─────────┐    显示3D模型/图片          ┌─────────┐
   │ A-Frame │ ──────────────────────────▶ │ 用户视野│
   │         │    叠加信息标签             │         │
   └─────────┘                              └─────────┘

7. 触发奖励
   ┌─────────┐    POST /api/points/add     ┌─────────┐
   │  前端   │ ──────────────────────────▶ │  后端   │
   │         │    {exhibitId, action}      │         │
   └─────────┘                              └─────────┘
```

### 6.4 AI对话数据流

```
┌─────────────────────────────────────────────────────────────────┐
│                        AI对话流程                                │
└─────────────────────────────────────────────────────────────────┘

1. 用户发送消息
   ┌─────────┐    POST /api/chat/send      ┌─────────┐
   │  前端   │ ──────────────────────────▶ │  后端   │
   │         │    {message, sessionId}     │         │
   └─────────┘                              └─────────┘

2. 获取历史上下文
   ┌─────────┐    GET chat:history:xxx     ┌─────────┐
   │  后端   │ ──────────────────────────▶ │  Redis  │
   │         │    获取最近N条对话          │         │
   └─────────┘                              └─────────┘

3. 构建AI请求
   ┌─────────┐    构建Prompt               ┌─────────┐
   │  后端   │ ──────────────────────────▶ │ 系统提示│
   │         │    + 历史上下文             │         │
   │         │    + 用户问题               │         │
   └─────────┘                              └─────────┘

4. 调用AI服务
   ┌─────────┐    POST /v1/chat/completions┌─────────┐
   │  后端   │ ──────────────────────────▶ │ 通义千问│
   │         │    stream: true             │  API    │
   └─────────┘                              └─────────┘

5. 流式响应
   ┌─────────┐    SSE data: xxx            ┌─────────┐
   │ 通义API │ ──────────────────────────▶ │  后端   │
   │         │    多次推送                 │         │
   └─────────┘                              └─────────┘

6. 转发前端
   ┌─────────┐    SSE data: xxx            ┌─────────┐
   │  后端   │ ──────────────────────────▶ │  前端   │
   │         │    实时推送                 │         │
   └─────────┘                              └─────────┘

7. 前端渲染
   ┌─────────┐    逐字显示                 ┌─────────┐
   │  前端   │ ──────────────────────────▶ │  UI     │
   │         │    打字机效果               │         │
   └─────────┘                              └─────────┘

8. 存储历史
   ┌─────────┐    RPUSH chat:history:xxx   ┌─────────┐
   │  后端   │ ──────────────────────────▶ │  Redis  │
   │         │    保存对话记录             │         │
   └─────────┘                              └─────────┘
```

### 6.5 数据展示方式

#### 6.5.1 地图展示

| 数据类型 | 展示方式 | 说明 |
|----------|----------|------|
| 景点位置 | 自定义Marker | 不同分类使用不同颜色 |
| 用户位置 | 脉冲动画圆点 | 带呼吸效果的定位标记 |
| 城市信息 | 信息卡片 | 切换城市时显示简介 |
| 景点详情 | 底部弹出卡片 | 包含图片、描述、操作按钮 |

#### 6.5.2 AR展示

| 数据类型 | 展示方式 | 说明 |
|----------|----------|------|
| 识别目标 | 相机叠加 | 实时识别框提示 |
| 3D模型 | A-Frame渲染 | 支持旋转、缩放交互 |
| 信息标签 | 悬浮文字 | 文物名称、朝代等 |
| 音频讲解 | 背景播放 | 自动播放解说音频 |

#### 6.5.3 列表展示

| 数据类型 | 展示方式 | 说明 |
|----------|----------|------|
| 收藏文物 | 网格卡片 | 图片+名称+朝代 |
| 徽章成就 | 横向滚动 | 图标+名称+进度 |
| 对话历史 | 时间线列表 | 头像+内容+时间 |

#### 6.5.4 详情展示

| 数据类型 | 展示方式 | 说明 |
|----------|----------|------|
| 文物图片 | 可点击放大 | 支持手势缩放 |
| 文物信息 | 卡片布局 | 名称、朝代、地点、描述 |
| 诗词内容 | 竖排文字 | 古风排版效果 |
| 阅历奖励 | 动画数字 | 获得时数字跳动 |
