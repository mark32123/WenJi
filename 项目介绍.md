# 文迹App项目计划书

## 一、项目概述

### 1.1 背景
在“文化自信”与“数字中国”战略推动下，非遗文化的年轻化传播亟需创新载体。然而，传统展示方式缺乏互动性与激励机制，难以吸引Z世代用户深度参与。“文迹”App以“行走+体验+奖励”为核心，尝试用轻量化技术手段降低文化接触门槛。

### 1.2 目的
- 打造一个基于地理位置的传统文化探索工具；
- 通过AR虚拟展示增强用户对非遗项目的直观理解；
- 建立“打卡—积累—兑换”闭环，提升用户持续使用意愿；
- 为后续接入文旅资源、教育场景打下产品基础。

### 1.3 创新点
- **文化地图点亮机制**：借鉴游戏化设计，用户每到访一个文化地标即可点亮，形成个人文化足迹；
- **AR沉浸式展示**：无需专业设备，手机即可查看非遗技艺的3D还原（如剪纸、皮影、古建结构）；
- **实用型激励体系**：通过体验行为赚取“文迹经验”，可兑换话费等现实奖励，提升转化率；
- **轻量级技术架构**：基于 Vue + JWT 快速构建 MVP（最小可行产品），便于迭代验证。

---

## 二、技术方案

### 2.1 总体架构
采用前后端分离模式，技术栈聚焦当前已掌握能力：
- **前端**：Vue 3 + Vite + Pinia（状态管理） + Vant（移动端UI），通过 H5 或混合开发（如 Capacitor）打包为 App；
- **后端**：Spring Boot（或其他 Java/Node.js 框架），提供 RESTful API；
- **认证机制**：基于 JWT（JSON Web Token）实现无状态登录，支持 token 刷新与过期处理；
- **地图服务**：集成高德地图 JavaScript API，实现定位与地理围栏（Geofencing）；
- **AR 功能**：使用 8thWall 或 AR.js 等 WebAR 方案（无需原生开发），加载 glTF 格式的轻量模型；
- **数据库**：MySQL 存储用户信息、点亮记录、AR内容元数据等。

> 注：暂未集成 TTS 语音播报，AI 讲解以图文形式呈现；未来可扩展接入大模型 API 生成动态讲解文案。

### 2.2 功能现状与规划

| 模块 | 当前完成 | 后续计划 |
|------|--------|--------|
| 用户系统 | ✅ 登录/登出（JWT）<br>✅ 修改个人信息<br>✅ 注销账户（逻辑删除） | 支持微信快捷登录 |
| 地图功能 | ✅ 基础定位（高德地图） | ✅ 接入文化地标数据库<br>✅ 实现“点亮地图”逻辑与可视化 |
| AR 展示 | ✅ 基础 WebAR 模型加载 | 优化模型加载性能，增加交互提示 |
| 激励体系 | ❌ 尚未开发 | 设计经验值规则，对接话费兑换接口（如运营商合作或第三方平台） |
| 内容跳转 | ❌ | 实现从 App 内跳转至非遗官网、预约页面等 |

### 2.3 关键技术（当前聚焦）
- **JWT 安全认证**：防止 token 泄露、支持自动续期；
- **WebAR 轻量化展示**：避免原生开发复杂度，快速验证 AR 可行性；
- **地理围栏触发**：当用户进入预设文化点半径（如200米），前端调用后端接口记录“点亮”行为；
- **响应式 H5 适配**：确保在 iOS/Android 主流机型上流畅运行。

### 2.4 技术限制与应对

| 限制 | 应对策略 |
|------|--------|
| 无原生 App 开发能力 | 采用 H5 + Capacitor / PWA 方案，兼顾跨平台与发布灵活性 |
| 无 TTS 语音功能 | 先以图文讲解为主，标注“语音版即将上线”引导期待 |
| 话费兑换依赖第三方 | 初期可用“积分排行榜+抽奖”替代，降低对接门槛 |

## 三、项目计划

### 3.1 可行性分析（基于当前条件）

| 维度 | 分析 |
|------|------|
| **技术可行性** | 已完成核心模块（登录、定位、AR展示），技术栈均为团队熟悉领域，无重大技术瓶颈。 |
| **开发成本** | 前端 Vue + 后端 Spring Boot 为常见组合，开源组件丰富，人力与时间成本可控。 |
| **用户价值** | “点亮地图 + 兑换话费”具备明确行为驱动力，易形成初期种子用户粘性。 |
| **风险点** | AR 在低端机性能不佳；话费兑换需商务合作。建议先做 MVP 验证核心玩法。 |

> **结论**：项目在现有技术条件下具备**高可行性**，建议优先完成“点亮地图 + 经验系统”闭环。

### 3.2 排期规划（2026年2月 – 2026年5月）

| 阶段 | 时间 | 交付物 |
|------|------|--------|
| **MVP 完善** | 2026.2 | - 修复登录/注销逻辑漏洞<br>- 优化 AR 加载速度<br>- 完成文化地标数据库（至少50个点位） |
| **核心玩法上线** | 2026.3 | - 实现“到达即点亮”功能<br>- 设计经验获取规则（如：点亮+10，AR互动+15）<br>- 前端展示“我的足迹”地图 |
| **激励机制试运行** | 2026.4 | - 接入模拟话费兑换（如：100经验 = 1元话费券）<br>- 内部测试兑换流程<br>- 增加“分享足迹”按钮 |
| **小范围公测** | 2026.5 | - 发布测试版（TestFlight / 蒲公英）<br>- 收集20+用户反馈<br>- 输出迭代路线图 |

## 四、结语
“文迹”立足于“小而美”的产品理念，不追求大而全，而是通过**一个清晰的文化动线（走到 → 看到 → 点亮 → 兑换）**，让用户在日常行走中自然接触传统文化。当前技术方案务实可行，后续将根据用户反馈逐步引入 AI 语音、社交裂变、文旅合作等能力。
